Data Collection Methodology

Data Source

All statistics are collected from the official VATSIM Data API (data.vatsim.net/v3/vatsim-data.json) which provides real-time network status updates.

Collection Frequency

Network snapshots are captured every 5 minutes via an Azure Logic App scheduler, providing approximately 288 data points per day.

Data Coverage

  • Start Date: September 5, 2021
  • Total Snapshots: 444,000+ and growing
  • Known Gap: May 15 - June 19, 2023 (36 days missing)

Metrics Collected

  • pilots - Total connected pilot clients
  • controllers - Total connected ATC positions
  • prefiles - Prefiled flight plans (not connected)
  • Derived: hour_of_day, day_of_week, month, season

Traffic Level Classification

Traffic levels are assigned using percentile-based baselines computed from historical data for each hour/day-of-week combination:

  • very_low: Below 10th percentile
  • low: 10th-25th percentile
  • below_average: 25th-50th percentile
  • average: 50th-75th percentile
  • above_average: 75th-90th percentile
  • high: 90th-95th percentile
  • very_high: Above 95th percentile

Regional Time Windows

Regional analysis uses UTC time windows as proxies for geographic activity:

  • APAC Hours (00-07 UTC): Asia-Pacific evening prime time
  • Europe Hours (08-15 UTC): European daytime
  • Americas Hours (16-23 UTC): Americas afternoon/evening

Note: These are approximations. VATSIM does not provide per-region user counts.
